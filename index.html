Da Vinci PAS FHIR ↔ ASC X12 278 Mapping Pipeline

Prior Authorization Support requires translating FHIR APIs to standard HIPAA X12 278 transactions for payer adjudication backend processing.

1

FHIR Request Bundle

Standard JSON payload sent via Claim/$submit operation containing clinical resources.

Patient.name
Practitioner.npi
Claim.item.productOrService
➔
2

ASC X12 278 Request

The FHIR server translates the payload into an X12 278 request transaction.

Loop 2010CA NM1*IL (Subscriber)
Loop 2010AA NM1*1Y (Billing Provider)
Loop 2000C SV1 Segment (Service Line)
➔
3

ASC X12 278 Response

The payer's core adjudication engine generates the HIPAA X12 278 response.

HCR01 = "A" (Approved) or "A3" (Pended)
HCR03 = Auth Ref Number
Loop 2000F HSD Segment (Quantity/Dates)
➔
4

FHIR ClaimResponse

The client receives a FHIR ClaimResponse resource with decision mapped back to JSON.

ClaimResponse.outcome
ClaimResponse.disposition
ClaimResponse.item.adjudication
